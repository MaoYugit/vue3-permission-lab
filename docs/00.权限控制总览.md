# 00. 前端权限控制总览

### 1. 页面/路由级别权限（Route-level）
这是最基本、最直观的权限控制，目的是**限制用户访问未授权的页面**。
*   **控制方式**：
    *   **动态路由（推荐）**：前端只配置不需要权限的公共路由（如登录页、404等）。用户登录后，前端向后端请求该用户拥有的菜单/路由数据，再利用路由库的动态添加 API（如 Vue Router 的 `router.addRoute`，或 React Router 的动态配置）将有权限的路由动态注入到路由表中。
    *   **路由守卫（拦截器）**：在路由跳转前进行全局拦截（如 `beforeEach`），检查目标路由是否在用户的权限列表中。如果无权访问，则重定向到 403 或登录页面。

### 2. 菜单/导航栏级别权限（Menu-level）
与路由权限相辅相成。用户登录后，左侧或顶部的导航菜单应当只展示该用户有权访问的模块。
*   **控制方式**：
    *   通常根据后端返回的路由表或权限列表，过滤出菜单树结构，再通过组件渲染出来。这样可以避免用户看到“看得见却点不进去”的菜单，提升用户体验。

### 3. 操作/按钮级别权限（Element-level）
即使在同一个页面内，不同角色的用户能进行的操作也可能不同。例如，普通用户只能“查看”，而管理员可以看到并操作“编辑”、“删除”和“新增”按钮。
*   **控制方式**：
    *   **自定义指令（如 Vue 指令）**：在 Vue 中，常通过自定义指令（如 `v-permission="['admin', 'editor']"`）来控制按钮的显示。指令在挂载时判断当前用户的权限，若不满足则直接移除该 DOM 节点或将其设置为 `disabled` 状态。
    *   **高阶组件/权限组件包装（如 React）**：在 React 中，常使用一个通用的权限包装组件（如 `<Authorized authority="user:delete">...</Authorized>`），不符合条件时返回 `null` 或禁用态的子组件。

### 4. 接口/数据请求级别权限（API-level）
前端的任何视图层控制都是可以被技术手段绕过的（例如通过控制台修改 DOM 或直接拼装 URL），因此**最关键的防线是接口层面的权限校验**。
*   **控制方式**：
    *   **请求拦截器**：在发送请求时，自动在 Request Headers 中携带身份凭证（如 `Authorization: Bearer <Token>`）。
    *   **响应拦截器**：对后端返回的错误状态码进行统一处理。例如，捕获到 `401`（未登录/Token过期）时跳转至登录页；捕获到 `403`（无权操作）时弹出友好提示，避免页面因异常数据崩溃。

### 5. 视图数据展示/脱敏级别权限（Data-level）
有些场景下，不同权限的用户看到的同一份数据，其展示细节不同。例如，普通客服只能看到脱敏后的电话号码（如 `138****8888`），而主管可以看到完整的号码。
*   **控制方式**：
    *   虽然敏感数据的脱敏工作必须由后端完成，但前端需要根据不同的权限字段，渲染不同的 UI 样式、占位符或格式化组件。
