# 04. 前端基础环境初始化与路由配置

本指南介绍如何在根目录 `vue3-permission-lab` 下初始化基于 **Vue 3 + TypeScript + Vite + Pinia + Element Plus** 的前端应用，并完成静态路由与项目骨架的搭建。

---

## 步骤 1：创建目录与 Vite 初始化

在项目根目录下创建并初始化前端项目。为了保持简洁，我们直接将文件放置于前端根文件夹。

```bash
# 确保当前在项目根目录 vue3-permission-lab 下
cd vue3-permission-lab

# 使用 Vite 初始化 Vue 3 + TS 模板
pnpm create vite frontend --template vue-ts

# 进入前端目录
cd frontend
```

---

## 步骤 2：安装依赖包

使用 `pnpm` 安装项目所需的运行依赖与开发依赖。

```bash
# 安装核心运行依赖
pnpm add vue-router@4 pinia pinia-plugin-persistedstate axios element-plus

# 安装开发依赖（Sass 预处理器、Node.js 类型声明以支持配置文件编写）
pnpm add -D sass eslint prettier @types/node
```

---

## 步骤 3：配置路径别名（`@`）

为了在项目中方便地使用绝对路径（如 `@/views/...`），我们需要同时修改 Vite 和 TypeScript 的配置。

### 1. 修改 `vite.config.ts`
配置别名指向 `src` 目录：

```typescript
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import path from 'path';

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:3000', // 转发到后端 Express 服务
        changeOrigin: true
      }
    }
  }
});
```

### 2. 修改 `tsconfig.app.json`（或 `tsconfig.json`）
在 `compilerOptions` 中添加路径解析映射，确保 TS 不会报路径找不到的错误：

```json
{
  "compilerOptions": {
    // ... 其他默认配置保持不变
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

## 步骤 6：配置路由与状态管理

### 1. 配置静态路由 `src/router/index.ts`
```typescript
import { createRouter, createWebHistory, RouteRecordRaw } from 'vue-router';

// 静态路由表：不需要任何权限即可访问的公开页面
export const constantRoutes: RouteRecordRaw[] = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/views/login/Login.vue'),
    meta: { title: '登录' }
  },
  {
    path: '/403',
    name: '403',
    component: () => import('@/views/error/403.vue'),
    meta: { title: '无权访问' }
  },
  {
    path: '/404',
    name: '404',
    component: () => import('@/views/error/404.vue'),
    meta: { title: '页面未找到' }
  },
  {
    path: '/',
    name: 'Layout',
    component: () => import('@/views/layout/Layout.vue'),
    redirect: '/dashboard',
    children: [
      {
        path: 'dashboard',
        name: 'Dashboard',
        component: () => import('@/views/dashboard/Dashboard.vue'),
        meta: { title: '控制台' }
      }
    ]
  },
  // 404 捕获路由
  {
    path: '/:pathMatch(.*)*',
    redirect: '/404'
  }
];

const router = createRouter({
  history: createWebHistory(),
  routes: constantRoutes
});

export default router;
```

### 2. 配置 Pinia 状态管理 `src/stores/index.ts`
配置 Pinia 并应用持久化插件：

```typescript
import { createPinia } from 'pinia';
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate';

const pinia = createPinia();
pinia.use(piniaPluginPersistedstate);

export default pinia;
```

## 步骤 7：装配入口文件 `src/main.ts`

将 Element Plus、路由、状态管理装配到 Vue 实例中。

```typescript
import { createApp } from 'vue';
import App from './App.vue';
import router from './router/index.js';
import pinia from './stores/index.js';

// 引入 Element Plus 样式
import ElementPlus from 'element-plus';
import 'element-plus/dist/index.css';

const app = createApp(App);

app.use(pinia);
app.use(router);
app.use(ElementPlus);

app.mount('#app');
```

## 步骤 8：启动与运行验证

在 `/frontend` 目录下运行：

```bash
pnpm dev
```

### 运行验证：
1.  浏览器默认打开 `http://localhost:5173/`，应当自动重定向至 `http://localhost:5173/dashboard`，展现后台的基础 `Layout` 架构与“控制台”内容。
2.  点击顶部右侧“退出登录”，应正常跳转到登录页面；在登录页面点击“登录”按钮，能够正常跳回首页。
3.  手动在地址栏输入 `http://localhost:5173/non-existent-page`（不存在的路由），系统应自动重定向至 404 错误页面。
