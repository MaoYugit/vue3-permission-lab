# 03. 后端基础环境初始化指南

本指南详细介绍如何从零搭建基于 **Express + TypeScript + ESM (ECMAScript Modules) + MongoDB (Mongoose)** 的后端开发环境。

---

## 步骤 1：创建目录与初始化 `package.json`

在项目根目录 `vue3-permission-lab` 下创建 `backend` 文件夹，并初始化 Node.js 项目。

```bash
# 进入根目录（若已在根目录则忽略第一步）
cd vue3-permission-lab

# 创建后端目录并进入
mkdir backend
cd backend

# 初始化 package.json
pnpm init
```

### 配置 `package.json` 为 ESM 模块规范
在执行 pnpm init 后，生成的是一个最基础的 Node.js 项目配置文件。为了满足我们使用 **TypeScript (TS)** 和 **ESM (ECMAScript Modules)** 的现代工程化开发需求，我们需要对其进行修改。：

```json
{
  "name": "backend",
  "version": "1.0.0",
  "description": "Vue3 Permission Lab Backend Service",
  "type": "module",
  "main": "dist/server.js",
  "scripts": {
    "dev": "tsx watch src/server.ts",
    "build": "tsc",
    "start": "node dist/server.js"
  },
  "packageManager": "pnpm@10.33.4"
}
```

1. 为什么要增加 "type": "module"？

- **默认状态**：Node.js 默认将所有 .js 文件识别为传统的 **CommonJS (CJS)** 模块，这意味着你只能使用 const express = require('express')。
- **修改理由**：加入 "type": "module" 告诉 Node.js：**本项目采用现代的 ECMAScript Modules (ESM) 规范**。这样我们就可以在项目中使用标准且更主流的 import express from 'express' 语法，使前后端代码模块规范保持一致。

2. 为什么要修改 "main": "index.js" 为 "main": "dist/server.js"？

- **默认状态**：Node.js 默认寻找根目录下的 index.js 作为项目入口。
- **修改理由**：
  1. 我们的源码是用 TypeScript 编写并放在 src/server.ts 中的，Node.js 无法直接在生产环境中运行 TS 文件。
  2. 在生产环境中，我们会通过 TypeScript 编译器（tsc）将 src 目录下的 TS 文件编译成原生 JS，并输出到 dist 目录中。
  3. 因此，编译后的**生产环境真正入口**是 dist/server.js，我们需要显式地将其指定为 main。

3. 为什么要修改 "scripts"（脚本命令）？

默认的 "test" 脚本只是一个占位符。我们将其替换为以下三个核心命令：

- **"dev": "tsx watch src/server.ts"**：
  - **作用**：在开发阶段运行。
  - **原理**：tsx 能够在内存中实时解析并运行 TypeScript，而不需要我们手动执行编译；watch 参数会监控 src 目录下所有文件的变化，一旦你保存代码，它就会自动重启服务。这为开发提供了极速的反馈。
- **"build": "tsc"**：
  - **作用**：在部署到生产环境之前运行。
  - **原理**：调用 TypeScript 编译器，根据 tsconfig.json 的规则将整个 src 目录下的 TS 源码安全地编译为标准的 ESM JavaScript，并放入 dist 文件夹。
- **"start": "node dist/server.js"**：
  - **作用**：在生产环境中运行。
  - **原理**：在服务器上，我们不再使用 tsx 这种开发工具，而是直接用原生的 node 运行编译好的 dist/server.js，保证服务的稳定与高效。

4. 关于其他元数据（keywords, author, license 等）

- 由于我们这是一个私有的、用于学习和训练的本地项目，默认生成的 keywords、author、license 等字段没有实质作用，可以予以删除以保持配置文件的整洁。
- 保留 "packageManager": "pnpm@10.33.4" 是一个很好的习惯，它锁定了当前项目必须使用 pnpm 进行包管理，防止其他协作人员误用 npm 或 yarn 导致依赖版本冲突。

## 步骤 2：安装依赖包

采用 `pnpm` 安装项目运行所需的生产依赖与开发依赖。为了避免 C++ 依赖库（`node-gyp`）带来的跨平台编译问题，密码哈希库推荐选用纯 JavaScript 实现的 `bcryptjs` [1]。

```bash
# 安装核心生产依赖
pnpm add express mongoose dotenv cors helmet jsonwebtoken bcryptjs

# 安装开发依赖及相关 TypeScript 类型定义
pnpm add -D typescript @types/node @types/express @types/cors @types/jsonwebtoken @types/bcryptjs tsx
```

---

## 步骤 3：配置 `tsconfig.json` (ESM 规范)

在 `/backend` 目录下创建 `tsconfig.json` 文件。为了在 Node.js 环境中顺利使用原生 `import/export`，我们需要将 `module` 与 `moduleResolution` 设置为 `NodeNext`。

```json
{
  "compilerOptions": {
    "target": "ES2022",                          /* 设定目标 JavaScript 版本 */
    "module": "NodeNext",                        /* 采用 Node.js 原生的 ESM 模块系统 */
    "moduleResolution": "NodeNext",              /* 设定模块解析策略以支持 ESM */
    "esModuleInterop": true,                     /* 允许使用 CommonJS 模块导入 */
    "strict": true,                              /* 开启严格类型检查 */
    "skipLibCheck": true,                        /* 忽略声明文件的类型检查 */
    "outDir": "./dist",                          /* 编译输出目录 */
    "rootDir": "./src"                           /* 源码根目录 */
  },
  "include": ["src/**/*"]
}
```

## 步骤 4：配置环境变量 (.env)

在 `/backend` 根目录下创建 `.env` 文件，用于管理数据库连接串、服务端口、JWT 密钥等敏感/多环境配置。

## 步骤 5：完成数据库连接与 Express 基础服务

创建源代码目录 `src`，并建立清晰的骨架。

```bash
mkdir -p src/config
```

### 1. 编写 MongoDB 连接配置 (`src/config/db.ts`)
利用 Mongoose 建立与本地 MongoDB 的连接。

```typescript
import mongoose from 'mongoose';
import dotenv from 'dotenv';

dotenv.config();

const MONGODB_URI = process.env.MONGODB_URI || 'mongodb://127.0.0.1:27017/vue3-permission-lab';

export const connectDB = async (): Promise<void> => {
  try {
    await mongoose.connect(MONGODB_URI);
    console.log('▶ [Database] MongoDB 连接成功');
  } catch (error) {
    console.error('▶ [Database] MongoDB 连接失败:', error);
    process.exit(1); // 连接失败则强制退出进程
  }
};
```

### 2. 封装 Express 应用实例 (`src/app.ts`)
将基础中间件（CORS 安全跨域、Helmet 安全响应头、JSON 解析）注入应用。

```typescript
import express, { Application, Request, Response } from 'express';
import cors from 'cors';
import helmet from 'helmet';

const app: Application = express();

// 安全与基础中间件
app.use(helmet());
app.use(cors({
  origin: 'http://localhost:5173', // 预留给 Vue3 前端本地服务的跨域配置
  credentials: true
}));
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// 基础健康检查接口
app.get('/api/health', (req: Request, res: Response) => {
  res.status(200).json({
    status: 'ok',
    timestamp: new Date().toISOString()
  });
});

export default app;
```

### 3. 编写启动入口文件 (`src/server.ts`)
负责连接数据库并拉起 HTTP 服务监听端口。

```typescript
import app from './app.js'; // 注意：在 ESM 规范下，TS 文件引入必须带有 .js 后缀
import { connectDB } from './config/db.js';
import dotenv from 'dotenv';

dotenv.config();

const PORT = process.env.PORT || 3000;

const startServer = async () => {
  // 1. 先建立数据库连接
  await connectDB();

  // 2. 监听端口
  app.listen(PORT, () => {
    console.log(`▶ [Server] Express 服务已启动，监听端口: http://localhost:${PORT}`);
  });
};

startServer();
```
